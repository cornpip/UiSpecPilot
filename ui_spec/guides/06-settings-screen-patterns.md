# 06. Settings Screen Patterns

`ui_spec`에서 `settings` 화면을 만들 때 쓰는 UI 패턴 가이드입니다.

이 문서는 옵션의 종류와 배치 방식을 시각적으로 정리하기 위한 기준입니다.
코드 구현 방식이나 실제 저장 구조를 정의하지는 않습니다.

## 목적

- 설정 화면이 잡다한 메뉴 모음처럼 보이지 않게 정리
- 각 설정 항목이 어떤 유형인지 UI만 봐도 구분 가능하게 설계
- `todo` 앱처럼 반복 사용되는 앱에서 피로감 없는 설정 화면 구성

## 권장 구조

기본적으로 아래 순서를 우선 고려합니다.

1. `app bar`
2. 상단 설명 또는 프로필 요약
3. 설정 그룹 2~4개
4. 하단 정보 또는 위험 액션

## 상단 영역

- 제목은 단순하게 `설정`, `앱 설정`, `환경 설정` 수준으로 둔다.
- 필요하면 한 줄 설명으로 화면 목적을 붙인다.
- 프로필이 핵심 앱이 아니면 상단 프로필 영역을 과하게 키우지 않는다.
- 상단 영역은 히어로 배너보다 정돈된 카드 또는 짧은 요약이 적합하다.

## 설정 그룹

- 관련 없는 항목을 한 그룹에 몰아넣지 않는다.
- 보통 `사용 경험`, `알림`, `표시 방식`, `지원/정보` 정도로 나눈다.
- 그룹 제목은 짧게 두고, 그룹 간 간격으로 구조를 먼저 읽히게 한다.

## 설정 행 유형

설정 행은 아래 4종류를 우선 패턴으로 사용합니다.

### Toggle Row

- 즉시 켜고 끄는 옵션에 사용한다.
- 행 오른쪽에는 switch 성격 UI를 둔다.
- 현재 상태는 시각적으로 즉시 읽혀야 한다.
- 예: `알림 받기`, `완료 항목 자동 숨기기`

### Select Row

- 현재값이 있고 다른 선택지로 바꾸는 옵션에 사용한다.
- 오른쪽에는 현재값과 chevron 또는 modal affordance를 둔다.
- inline dropdown보다 바텀시트/선택 패널을 여는 흐름이 모바일에 더 적합하다.
- 예: `기본 정렬`, `언어`, `시작 화면`

### Navigation Row

- 다른 화면으로 진입하는 옵션에 사용한다.
- 오른쪽에는 chevron만 두고 현재값은 생략 가능하다.
- 예: `알림 시간 설정`, `태그 관리`, `도움말`

### Destructive Row

- 로그아웃, 초기화, 삭제 같은 낮은 빈도 위험 행동에 사용한다.
- 일반 항목과 그룹 분리 또는 색상 분리로 위험도를 명확히 한다.
- 한 화면에서 primary action처럼 보이지 않게 한다.

## 텍스트 위계

- 1행 제목은 가장 먼저 읽혀야 한다.
- 필요하면 2행 설명으로 영향 범위를 짧게 보완한다.
- 설명 문구가 길어지면 설정 이해가 느려지므로 짧게 유지한다.
- 현재값은 제목보다 덜 강하게 보이되, 충분히 읽혀야 한다.

## 시각 밀도

- 설정 화면은 과한 카드 중첩보다 일정한 row rhythm이 중요하다.
- 한 그룹 내 행 높이는 가능하면 통일한다.
- 아이콘은 꼭 필요할 때만 쓰고, 텍스트만으로도 구분되면 생략 가능하다.
- divider, 카드, 배경 구획 중 한두 가지 방식만 선택해 일관되게 간다.

## Todo App Recommended Items

별도 요구가 없으면 `todo` 앱 설정의 기본 후보는 아래 정도로 본다.

- 알림 받기
- 오늘 할 일 우선 정렬
- 완료한 일 숨기기
- 기본 시작 화면
- 언어
- 도움말 / 앱 정보

## 작성 시 체크 질문

- 각 행이 toggle, select, navigation, destructive 중 무엇인지 바로 보이는가?
- 현재값이 필요한 행에 실제 현재값 표시가 있는가?
- 위험 액션이 일반 설정과 시각적으로 분리되는가?
- 그룹 제목만 읽어도 화면 구조가 이해되는가?
