# 07. App Bar And EndDrawer Patterns

`ui_spec`에서 공통 상단 구조와 `endDrawer`를 만들 때 참고하는 UI 패턴 가이드입니다.

이 문서는 shell 코드 구조가 아니라
사용자가 화면 상단과 보조 메뉴를 어떻게 읽게 할지에 대한 기준입니다.

## 목적

- 화면마다 제각각인 상단 구조를 줄이고 일관된 맥락 전달 유지
- `endDrawer`를 메인 콘텐츠를 방해하지 않는 보조 탐색 UI로 정리
- 홈, 상세, 설정 화면 사이의 공통 shell 감각 유지

## App Bar Core Role

- 현재 위치를 설명한다.
- 사용 가능한 핵심 보조 행동을 보여준다.
- 너무 많은 컨트롤을 몰아넣지 않는다.

## App Bar Recommended Slots

기본적으로 아래 3영역으로 해석합니다.

1. Leading
2. Title area
3. Trailing actions

## Leading Area

- 홈 화면에서는 메뉴, 프로필, 상태 표시 중 하나를 둔다.
- 상세/설정 화면에서는 back affordance를 우선한다.
- leading 아이콘은 장식보다 내비게이션 의미가 명확해야 한다.

## Title Area

- 제목은 현재 화면의 목적을 한눈에 이해시키는 문구여야 한다.
- 필요하면 한 줄 보조 설명을 둘 수 있지만, 과도하게 길어지지 않게 한다.
- 홈 화면에서는 `오늘`, `내 할 일`, `프로젝트` 같은 현재 맥락이 적합하다.
- 상세 화면에서는 항목명이나 섹션명이 직접 드러나는 편이 낫다.

## Trailing Actions

- 1~2개 정도만 두는 방향을 우선한다.
- 검색, 추가, 필터, drawer 오픈처럼 빈도 높은 보조 행동에 한정한다.
- 동등한 중요도를 가진 아이콘을 3개 이상 나열하지 않는다.

## Visual Tone

- `app bar`는 별도 세계처럼 보이지 않고 body와 연결되어야 한다.
- 지나치게 두꺼운 경계나 강한 그림자보다 여백과 타이포로 구분하는 편이 안정적이다.
- 홈과 설정에서 상단 높이가 너무 다르면 같은 앱 느낌이 약해질 수 있다.

## EndDrawer Core Role

- 메인 화면을 떠나지 않고 보조 이동과 보조 상태를 확인하는 곳이다.
- 주요 작업 생성이나 상세 편집보다 한 단계 바깥의 기능을 둔다.
- 자주 쓰는 핵심 작업을 drawer로 숨기지 않는다.

## EndDrawer Recommended Layout

기본적으로 아래 순서를 권장합니다.

1. 상단 요약
2. 메뉴 리스트
3. 보조 섹션
4. 하단 정보

## 상단 요약

- 앱 이름, 사용자 이름, 현재 모드, 필터 상태 같은 짧은 정보가 적합하다.
- 큰 프로필 배너보다 작은 요약 카드나 간단한 헤더가 모바일에서 안정적이다.

## 메뉴 리스트

- 항목 수는 너무 많지 않게 유지한다.
- 기본 후보는 `설정`, `완료한 일`, `보관함`, `태그`, `도움말` 정도다.
- 현재 위치 또는 현재 선택된 메뉴는 배경/텍스트/인디케이터로 구분한다.

## 보조 섹션

- 진행률, 동기화 상태, 작은 통계 같은 부가 정보를 배치할 수 있다.
- 이 영역은 요약적이어야 하며 본문보다 더 중요해 보이면 안 된다.

## 하단 정보

- 버전, 정책, 로그아웃 같은 낮은 빈도 요소를 둔다.
- 시각적 무게를 낮춰 메인 메뉴와 경쟁하지 않게 한다.

## Todo App Interpretation

`todo` 앱에서의 기본 해석은 아래를 우선합니다.

- 홈 `app bar`: 오늘 날짜, 화면 제목, 검색 또는 drawer 오픈
- 상세 `app bar`: back, 항목 제목, 편집 또는 더보기
- 설정 `app bar`: back, 단순 제목
- `endDrawer`: 설정 진입, 완료 목록, 보관 목록, 도움말, 앱 상태 요약

## 작성 시 체크 질문

- 상단만 봐도 현재 화면 맥락이 드러나는가?
- trailing action이 실제 자주 쓰는 보조 행동만 남아 있는가?
- drawer에 핵심 메인 작업이 과하게 숨어 있지 않은가?
- drawer 안에서 현재 선택 상태가 시각적으로 보이는가?
